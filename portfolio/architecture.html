Triangle Test Protocol

A three-validator consensus protocol implementing a four-gate validation framework for ensuring output quality and integrity.

Four-Gate Validation

  1. Gate 1 (Meta-Analysis): Guardian validates sovereignty preservation
  2. Gate 2 (Routing): Guardian validates ACL compliance
  3. Gate 3 (Payload): Parallel validation
    • 3a: Perplexity validates factual accuracy
    • 3b: Claude validates structural coherence
  4. Gate 4 (Audit): Guardian validates evidence chain integrity

Validation Modes

  • Standard: Allows minor variation during content cycles
  • Strict: Flags any semantic deviation
  • Custom: User-defined acceptable ranges

Guardian Thresholds

  • Low: Basic drift detection
  • Medium: Balanced sensitivity
  • High: Catches subtle inconsistencies (may generate false positives)

Four-Gate Authorization

Security and access control mechanisms for the CYW architecture.

Model Access Controls

Define which VLANs can access specific project contexts. Prevents unintended model exposure to sensitive IP or creative direction.

Output Logging

Tracks all model responses for audit trail. Required for patent documentation and IP protection.
